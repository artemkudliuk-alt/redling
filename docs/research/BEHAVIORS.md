# Анализ поведения и интерактива (Behaviors Specification)

Этот документ описывает интерактивные элементы, состояния и адаптивное поведение оригинального сайта и планируемые надстройки для проекта лендинга **Redling Hotel**.

---

## 1. Сайдбар слева (Left Sidebar)
Сайдбар является сквозным элементом интерфейса и имеет фиксированное положение.

### Десктопное поведение (экраны > 991px)
* **Позиционирование**: `position: fixed; left: 0; top: 0; width: 285px; height: 100vh;`
* **Граница**: Правая граница `1px solid rgba(77, 87, 96, 0.1)`.
* **Фон**: В оригинале используется фоновое изображение `side.jpg` (текстура белой бумаги или деликатный паттерн) с белым цветом подложки.
* **Логотип**: Размещен вверху, ведет на главную страницу.
* **Навигация**: Вертикальный список ссылок. При наведении на пункт меню происходит плавное изменение цвета текста с `#4d5760` на фирменный бирюзовый `#00aba9` (transition: `color 0.3s ease`).
* **Кнопка «★ АКЦИИ ★»**: Выделена зеленым/бирюзовым фоном, белым текстом и звездочками. Анимируется при наведении (эффект легкого увеличения или изменения яркости).
* **Контакты (Выпадающее меню телефонов и мессенджеров)**:
  * Клик/наведение на блок номера телефона раскрывает абсолютный блок `.drop` со списком мессенджеров (Viber, WhatsApp, Telegram).
  * Плавное раскрытие через CSS transitions (изменение высоты/прозрачности).
* **Социальные сети**: Иконки Facebook, Instagram, TripAdvisor в нижней части сайдбара. При наведении меняют прозрачность или цвет.
* **Языковой переключатель**: Ссылки `en`, `ru`, `ua`. Текущий активный язык выделен классом `current` (бирюзовый цвет и отсутствие подчеркивания).

### Адаптивное поведение (экраны < 991px)
* Сайдбар скрывается (`display: none` или трансформируется).
* Сверху появляется мобильная шапка (Header) с логотипом, телефоном и гамбургер-меню (кнопка меню).
* При клике на гамбургер-меню сайдбар выезжает слева (`transform: translateX(0)`) с плавным оверлеем (overlay) на всю оставшуюся часть экрана.

## 2. Скролл-видео фонового контента (Scroll-driven Video Background)
Для правой части лендинга применяется схема скролл-видео (Video Scrubbing).

* **Позиционирование плеера**:
  ```css
  .video-background-container {
    position: fixed;
    top: 0;
    left: 285px;
    width: calc(100% - 285px);
    height: 100vh;
    z-index: 1;
    overflow: hidden;
  }
  .video-background-container video {
    width: 100%;
    height: 100%;
    object-fit: cover;
  }
  ```
* **Затемнение**: На плеер накладывается полупрозрачная маска (overlay) для обеспечения читаемости белого текста поверх видео-кадров (например, `background: rgba(0, 0, 0, 0.45);`).
* **Управление прокруткой**:
  * Мы рассчитываем общий процент прокрутки документа: `scrollFraction = window.scrollY / (document.documentElement.scrollHeight - window.innerHeight)`.
  * Время видео обновляется на основе процента: `video.currentTime = video.duration * scrollFraction`.
  * Для предотвращения дерганий используется сглаживание воспроизведения через интерполяцию (Linear Interpolation - Lerp) внутри цикла `requestAnimationFrame`:
    ```javascript
    let targetTime = 0;
    let currentTime = 0;
    
    // В обработчике скролла:
    targetTime = video.duration * scrollFraction;
    
    // В цикле анимации:
    currentTime += (targetTime - currentTime) * 0.1; // коэффициент сглаживания 0.1
    video.currentTime = currentTime;
    ```

## 3. Секции контента (Foreground Content Sections)
Секции накладываются поверх видео-фона с более высоким `z-index: 2` и скроллятся стандартным образом.

* **Эффект появления**: При прокрутке и входе секции в область видимости (Intersection Observer), контент плавно проявляется (opacity: `0 -> 1`, transform: `translateY(30px) -> translateY(0)` с transition `all 0.8s cubic-bezier(0.25, 1, 0.5, 1)`).
* **Стилизация блоков**: Для блоков с текстовым описанием применяется стиль Glassmorphism (слегка размытый полупрозрачный белый или темный фон `backdrop-filter: blur(10px)`), чтобы текст хорошо читался поверх видео.
